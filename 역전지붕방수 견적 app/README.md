# 역전지붕방수 견적 앱 v4 (안도공간)

실제 엑셀 견적서(`동삭동 역전지붕방수 견적서 260630.xlsx`)를 **단일 `index.html`** React 앱으로 재현하고,
로그인·팀 공유·자재 단가 DB(v2), **구역별 자재선택**(v3)에 이어 **가로(A4) 인쇄/PDF·오픈 가입·Vercel 배포**(v4)까지 더한 버전입니다. 계산 결과는 엑셀과 **1원 단위(±1)** 로 일치합니다.

## 실행 방법 (2가지)

### 1) 서버 모드 — 로그인 + 팀 공유 (권장)
```bash
cd '역전지붕방수 견적 app'
npm install
npm start          # → http://localhost:3015
```
- 브라우저에서 **http://localhost:3015** 접속 → 로그인/가입 화면.
- **가입**: 이메일 · 비밀번호(8자 이상) · 이름. (v4: 초대코드 없음)
- 로그인하면 견적 목록·저장·자재 단가가 **서버(팀 전체 공유)** 로 오갑니다. 견적 편집은 자동 저장(약 0.8초 디바운스, 헤더에 `● 저장됨` 표시).

### 2) 로컬 모드 — 서버 없이 단일 파일 (오프라인)
- `index.html` 을 브라우저에서 **더블클릭**(`file://`) 하거나, `npx serve .` 같은 정적 서버로 열기.
- 서버(`/api/health`)에 접속되지 않으면 자동으로 **로컬 모드**로 전환됩니다(상단에 `로컬 모드` 배너).
- 로그인 UI 없이 동작하며, 견적/단가는 이 브라우저 **localStorage** 에 저장됩니다.
- 서버 모드로 처음 로그인할 때 로컬에 저장된 견적이 있으면 **"서버로 업로드"** 를 1회 제안합니다.

## 탭 구성 (직원용 / 고객 제출용 분리)

**직원용**
- **📝 견적입력** — ①프로젝트정보(유입경로 = 셀렉트: 티푸스/잡자재/패시브협회/기타) ②구역별 치수(**최대 10구역** · 마감/벽체마감 컬럼 없음) ③자재선택 ④가설공사 ⑤추가공사(체크 시 **수량·단가·할증 인라인 편집** + 직접입력) ⑥요율·할증·현장여건·특이사항. 우측 **실시간 요약 패널** + 📐 옥상 형태 계산기.
  - `☑️ 전체 마감방식 동일` **체크(기본)** → 공통 마감/벽체마감 + ③자재선택 카드 **1개**.
  - **해제** → 구역별 마감/벽체마감 + `③ 자재선택 — <구역명>` 카드를 **구역 수만큼**. 각 구역이 자기 자재·마감으로 계산되며, (아이템×변형)별 수량은 반올림 전에 합산돼 한 행으로 병합됩니다. **전 구역 동일 선택이면 체크 상태와 완전 동일한 수치**.
- **💰 자재DB** — ITEM_DB·장비 전 항목의 기본단가 옆에 **오버라이드 입력칸**. 서버 모드=팀 공유, 로컬 모드=localStorage. `●` = 수정됨.
- **📦 발주서** — 판매처별 그룹 · 포장규격 · 발주수량(=ROUNDUP(시공수량/제수)) · 원단가(할증 前). (직원용 탭이라 자체 🖨 인쇄 버튼 유지)
- **🗂 저장목록** — 견적별 **작성자·수정자·메모(인라인 편집)·총합계** · 새 견적/복제/삭제/불러오기 · 견적 JSON 내보내기·가져오기 · 동삭동 샘플 · 🧪 자가검증.

**고객 제출용** (개별 🖨 인쇄 버튼 없음 — 헤더 버튼으로 출력)
- **📄 견적서** — 표지 + 공종별 요약 + 공사원가계산서(부자재 행 포함) + 평당단가. 조건 문구 **4줄**(4. 본 견적서는 한달간 유효합니다.).
- **📋 내역서** — 4개 공종별 활성 항목 + 소계.
- **📖 자재소개** — 선택 자재별 카드(역할·특징·비고 + **이미지**) + 견적 설명 + 현장 특이사항.

## 헤더 [🖨 인쇄 · PDF]
- **① 표지+견적서 → ② 내역서 → ③ 자재소개** 순서로 결합된 **가로(A4 landscape)** 문서를 만들어 브라우저 인쇄 대화상자를 엽니다(발주서·직원용 화면은 제외).
- **PDF 저장**: 대화상자의 대상에서 **"PDF로 저장"** 선택 — 인쇄 품질(벡터 텍스트) 그대로 저장됩니다. (v4: 래스터 방식 html2pdf 내보내기 제거)
- 견적 JSON 내보내기/가져오기는 저장목록 탭에 별도 유지.

## v4 변경점 (v3 대비)
1. **내보내기 → 인쇄와 통합**: 헤더 버튼 하나(`🖨 인쇄 · PDF`). 결합 문서가 **가로(A4)** 로 바뀌어 표가 시원하게 나오고, "PDF로 저장" 결과물이 인쇄 미리보기와 동일(벡터).
2. **견적서에서 평당 단가 제거** — 견적입력 우측 요약(내부용)에만 표시.
3. **자재소개 이미지 file:// 폴백** — `manifest.json` fetch 가 막히는 환경(더블클릭 실행)에서도 내장 매니페스트로 이미지 표시.
4. **가입 초대코드 제거** — 이메일/비밀번호/이름만으로 가입.
5. **Vercel 배포 지원** — `vercel.json`(server.js 서버리스 + 정적 index.html/images), `server.js` 는 직접 실행 시에만 listen.

## v3 변경점 (v2 대비)
1. **⬇ 내보내기 = PDF 다운로드**(구 독립형 HTML 내보내기 제거). `html2pdf.js` 지연 로드 · 기존 `.page-break` CSS 연동.
2. 고객 제출용 3탭의 **개별 인쇄 버튼 제거**(헤더 인쇄만).
3. **유입경로**: 자유 입력 → **셀렉트**(빈 값·티푸스·잡자재·패시브협회·기타. 목록 밖 저장값은 임시 옵션으로 보존).
4. **구역별 자재선택**: `zones[i].sel` 도입. `전체 마감방식 동일` 해제 시 구역마다 자재·마감을 따로 지정 → 변형별로 행이 분리되고 동일 변형은 병합. 구역 테이블의 마감 컬럼은 양쪽 모드 모두 제거.
5. **견적서 조건 4번째 문구** 추가.

> **마이그레이션**: (v1→v2) `subInNetCost` 없으면 `true`, `finishAll` 없으면 `enabled:false`, 추가공사 `'예/아니오'`→객체. (→v3) `zones[].sel` 없으면 `est.sel` 딥카피로 생성하되 **ea 3필드(우레탄실리콘·트렌치)는 zones[0]에만** 두어 동삭동 총합계 불변, `coverNotes` 가 구 기본 3줄이면 4줄로 교체(커스텀 보존).

## 자재 이미지 (선택)
`images/materials/manifest.json` = `{ "<imgKey>": "<파일명>" }`. 자재소개 카드는 매니페스트에 있는 키만 `images/materials/<파일명>` 을 렌더하고, 없으면 이미지 없이 표시합니다(파일 부재가 앱을 깨지 않음). fetch 가 불가한 `file://` 실행에서는 내장 폴백 매니페스트(`<imgKey>.jpg` 규약)를 사용합니다.

## 구조 메모
- 계산 엔진은 `<script id="calc-engine">`(플레인 JS, `window.RoofCalc`)에 분리 — React(UMD + Babel **classic** runtime + Tailwind CDN)는 이 전역을 소비만 합니다. 계산 중간 반올림 없이 표시할 때만 반올림.
- v3 엔진: `computeEstimate(est, overrides?)` 는 `finishAll.enabled=false` 이고 구역이 `sel` 을 가지면 **구역별 병합 경로**(전역 단일패스와 분기), 그 외는 v2 그대로. `RoofCalc.withZoneSel(est)`·`RoofCalc.migrateCoverNotes(arr)` 노출.
- 백엔드(`server.js`, PORT 3015): Express + pg(Supabase) + bcryptjs + JWT, `roof_users/roof_estimates/roof_prices` 3테이블. `.env`(`DATABASE_URL`/`JWT_SECRET`)는 커밋 제외. Vercel 배포: `vercel.json` 참조(환경변수 동일 2개).
