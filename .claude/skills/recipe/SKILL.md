# 🍳 냉장고 레시피 생성 스킬

> **⭐ 가장 먼저 할 일 — `ingredients/` 폴더의 JSON 파일들을 전부 읽어라.**
> 레시피를 상상으로 지어내지 말고, **폴더에 실제로 있는 재료에서 출발**한다.

핵심 파이프라인:
`ingredients/*.json 먼저 읽기 → 재료 파악 → (기본 조건에 맞춰) 레시피 설계 → recipes/*.md 저장`

## 🎯 기본 조건 (별도 지시가 없으면 항상 적용)

- 🍽️ **1인분 기준** — 혼자 먹을 양으로 계량
- ⏱️ **15분 이내** — 준비 + 조리를 합쳐 빠르게
- 🔥 **자취생 난이도** — 냄비/프라이팬 1개, 흔한 양념, 복잡한 손질·특수 도구 없이

> 이 조건은 자유 텍스트 인자로 덮어쓸 수 있다. 예: `/recipe 2인분`, `/recipe 정성껏`, `/recipe 매콤하게`, `/recipe 30분 코스요리`.

## 입력 (인자)

- **인자 없음** — 위 기본 조건(1인분·15분·자취생)으로 레시피 생성
- **자유 텍스트** — 선호/조건 반영 (예: `/recipe 매콤하게`, `/recipe 야식`, `/recipe 2인분`, `/recipe 다이어트`)
- **`path=<경로>`** — 재료 폴더를 직접 지정 (예: `/recipe path=week-5/fridge-recipe/ingredients`)

## 데이터 규약 — 재료 1개 = JSON 파일 1개

각 재료 파일은 다음 스키마의 JSON 1개다:

```json
{ "name": "계란", "quantity": "6개", "category": "냉장" }
```

- `name` — 재료명 (한글)
- `quantity` — 수량 (예: "6개", "200g", "1/2포기")
- `category` — 보관 구분: `냉장` | `냉동` | `실온`

> 💡 **새 재료를 추가하려면 JSON 파일 하나만 만들면 된다 — 이 스킬 코드는 절대 수정하지 않는다.**
> 예: `ingredients/tofu.json` 파일만 추가하면, 다음 `/recipe`부터 두부가 자동으로 후보 재료에 포함된다. 삭제도 파일 하나만 지우면 끝.

## 절차 (이 순서대로 수행)

### 1) 재료 폴더 찾기
다음 우선순위로 `ingredients/` 디렉터리를 정한다:
1. 인자에 `path=`가 있으면 그 경로
2. 현재 폴더의 `./ingredients`
3. `week-5/fridge-recipe/ingredients`
4. 위가 모두 없으면 Glob `**/ingredients/*.json` (단, `node_modules` 제외)로 탐색해 가장 적절한 폴더 선택
- 사용한 재료 폴더 경로를 한 줄로 사용자에게 알린다.

### 2) ingredients/ JSON 먼저 읽기  ⭐
- **무엇을 만들지 정하기 전에**, Glob으로 그 폴더의 `*.json`을 **전부** 찾고 각 파일을 Read 해서 파싱한다.
- `{ name, quantity, category }`를 모은다. 파싱 실패한 파일은 건너뛰되 어떤 파일인지 경고한다.
- **재료가 0개면**: 레시피를 만들지 말고, 위 JSON 스키마 예시와 함께 "`ingredients/` 폴더에 재료 파일을 먼저 추가하세요"라고 안내하고 종료한다.
- 찾은 재료를 **표**로 요약한다 (재료 / 수량 / 보관).

### 3) 레시피 설계 (기본 조건 준수)
읽어 온 보유 재료를 **최대한 활용**하는 현실적인 레시피를 설계한다. **별도 지시가 없으면 1인분 · 15분 이내 · 자취생 난이도**를 지킨다.
- 보유 재료만으로 가능한 요리를 우선한다.
- 소금·기름·물·후추 같은 **기본 양념은 제외**하고, 추가로 사야 하는 재료는 따로 표시한다.
- `quantity`를 보고 **1인분에 맞게** 사용량을 정한다 (남는 재료는 그대로 둔다).
- 조리 단계는 **자취생이 15분 안에** 끝낼 수 있게 간결하게 (가능하면 4~6단계).
- 자유 텍스트 인자가 있으면(매콤/야식/인분/시간 등) 그 조건을 우선 반영한다.

### 4) 마크다운 작성
아래 구조로 작성한다 (메타에 조건을 명시):

```markdown
# {요리 이모지} {요리명}

> 🍽️ 1인분 · ⏱️ 약 N분 · 🔥 자취생 난이도

{한 줄 소개}

## 🧺 재료
- ✅ {보유 재료} {1인분 사용량}
- 🛒 {추가로 필요한 재료} (선택/필수 표시)

## 👩‍🍳 조리 순서
1. ...
2. ...

## 💡 팁
- ...

---
*생성일: {YYYY-MM-DD} · 조건: 1인분·15분·자취생 · 사용한 냉장고 재료: {재료 목록}*
```

### 5) 파일로 저장
- 출력 폴더: 재료 폴더와 같은 위치의 `recipes/` (없으면 생성).
- 날짜는 Bash `date +%F`로 **실제 값**을 구한다.
- 파일명: `recipe-{요리명-영문-slug}-{YYYY-MM-DD}.md`.
  - 같은 이름이 이미 있으면 `-2`, `-3` … 접미사로 **덮어쓰지 않는다**.
- 저장 후 **저장된 파일 경로**와 레시피 핵심(요리명·인분·예상시간)을 사용자에게 보고한다.

## 주의사항
- **항상 `ingredients/`를 먼저 읽고**, 거기 있는 재료에서만 출발한다.
- 기본 조건(1인분·15분·자취생)을 지키되, 사용자 인자가 있으면 그쪽을 우선한다.
- 기존 레시피 파일을 **덮어쓰지 않는다**.
- 재료가 빈약하면 솔직히 말하고, 추가하면 좋은 재료를 1~2개 제안한다 (단, 추가는 JSON 파일 1개면 된다는 점도 안내).
- 재료의 `category`(냉장/냉동/실온)와 수량을 레시피에 자연스럽게 반영한다 (예: 냉동 재료는 해동 단계 포함).
