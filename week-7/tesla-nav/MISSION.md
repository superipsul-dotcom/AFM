# 🎯 APP MISSION

> ✅ **확정본 (2026-07-14).** 차량 보유 여부와 AI의 역할이 확정되어 v0 초안의 미결 항목이 모두 닫혔습니다.

## Project Name

**안도 드라이브 (Ando Drive) 🚗**
*(폴더: `week-7/tesla-nav/` — 대안 이름: "테슬라 목적지 비서")*

브랜드 패밀리(안도공간 · 카페 안도 · 안도 빈즈)와 결이 맞는 이름으로 제안합니다.

## Mission Statement

**말 한 줄이면, 차에 타기 전에 목적지가 이미 내 테슬라 화면에 떠 있다 — 정확한 주소를 몰라도.**

## Problem Statement

목적지를 테슬라에 넣으려면 지금은 ① 네이버/카카오맵에서 정확한 상호·주소를 먼저 찾고 ② 공유하거나 ③ 차에서 직접 타이핑해야 한다. 테슬라 내비 검색은 한국 상호·주소에 약해서 결국 사람이 "정답 주소"를 미리 알아낸 뒤에야 시작할 수 있다.

진짜 문제는 전송이 아니라 **결정**이다. 머릿속에 있는 건 "오늘 가야 하는 현장", "성수동 조용한 카페" 같은 말이지 주소가 아니다. 지금은 그 말을 주소로 바꾸는 일을 매번 사람이 손으로 한다.

## Target Users

**1차 사용자 = 나 (안도공간 대표).** 하루에 여러 인테리어 현장을 차로 이동하는 테슬라 오너. 목적지가 매일 바뀌고, 그 목적지는 이미 내 현장 관리 앱(`interior-cost`) 안에 데이터로 존재한다.

공개 배포·타인 사용은 v1 대상이 아니다.

## 🔍 시장 현실 (이 미션의 근거)

디스커버리 중 확인된 사실 — **스코프를 여기에 맞춰야 합니다.**

- **"테슬라에 목적지 보내기" 자체는 이미 해결된 문제입니다.** 한국에만 [NaviToTesla](https://github.com/zipizigi/NaviToTesla)(오픈소스·안드로이드), [테슬라도](https://www.teslado.com/), [TESLINK](https://apps.apple.com/kr/app/teslink-share-to-ev/id6748283617)(iOS)가 이미 있고 잘 돕니다.
- 👉 따라서 **전송 파이프를 잘 만드는 것은 차별점이 0입니다.** 이미 있는 앱을 쓰면 됩니다.
- 👉 **이 앱의 존재 이유는 "AI가 목적지를 결정해 주는 부분" 하나뿐입니다.** 기존 앱들은 전부 "사용자가 이미 찾아낸 주소"를 받아서 넘겨줄 뿐, **내 현장 일정을 모릅니다.** 그게 우리가 이길 수 있는 유일한 지점입니다.
- 이 기존 앱들의 존재는 좋은 소식이기도 합니다 — **한국에서 실제로 된다는 것이 증명됐다**는 뜻이니까요.

## Core Features (v1)

1. **한 줄 자연어 → 목적지 해석** — "오늘 현장", "지난주 갔던 자재상", "논현동 타일집" 같은 말을 받아 AI가 후보 1~3개로 좁혀 제시
2. **확정 → 내 차로 전송** — 후보를 고르면 실제 테슬라 화면에 목적지가 뜬다 (이 앱의 최종 증명)
3. **내 현장·일정 기반 추천** — "오늘 어디 가?" 한마디에 `interior-cost`의 현장·일정을 읽고 목적지 제안. **기존 앱이 절대 못 하는 부분 = 차별점의 핵심.** 기능 1과 별개 화면이 아니라 같은 한 줄 입력의 특수 케이스로 구현한다.
4. **시뮬레이터 모드** — 실차 연동 없이도 "무엇이 전송될지"(이름·주소·좌표)를 화면에서 확인. 차량 연동이 막혀도 앱은 검증 가능한 상태를 유지
5. **최근·즐겨찾기 원탭 재전송** — 자주 가는 현장은 말도 필요 없이 한 번의 탭

## Anti-Scope (v1에서 하지 않을 것)

- ❌ **턴바이턴 경로 안내** — 테슬라가 하는 일. 우리는 "목적지를 꽂아주는 것"까지만.
- ❌ **주행 중 조작** — 타기 전 / 주차 중 사용 전제. 안전 문제이자 스코프 방어선.
- ❌ **내비 외 차량 제어** (충전·공조·도어 등) — 재밌어 보여도 문제와 무관.
- ❌ **경유지 다중 최적화** — v1은 목적지 1개. (여러 현장 순회 최적화는 v2 후보로 명시적 보류)
- ❌ **주행 기록·연비 분석** — 테슬라도/Tessie의 영역. 우리 문제 아님.
- ❌ **타인 배포·다중 사용자·과금** — 내 차, 내 계정 하나로 충분.

## ✅ 확정된 결정

### #1. 차량 — **보유 중, 2021년 이전 차량**
- **실차에 목적지가 뜨는 것이 v1 완료 조건입니다.** 위 성공 지표 표가 그대로 유효합니다.
- 2021년 이전 차량이라 **서명된 명령(Vehicle Command Protocol) 게이트를 피해 갑니다.** 연동 경로가 한 단계 단순해지는 것이고, 이 미션에서 가장 큰 기술 리스크 하나가 줄었습니다.
- 시뮬레이터 모드는 v1에서 빠지지 않습니다 — 실차 없이 개발·테스트하는 수단이자, 연동이 막혔을 때의 안전망으로 남깁니다.

### #2. AI의 역할 — **둘 다 (자연어 해석 + 현장·일정 자동 추천)**
- 컨텍스트 자동형을 자연어 해석의 **특수 케이스**로 구현합니다. "오늘 현장"도 결국 한 줄 자연어이므로 입력 경로는 하나입니다.
- 따라서 **`interior-cost`의 현장·일정 데이터를 읽는 것이 v1 범위에 포함됩니다.**

### #3. 위치 — `week-7/tesla-nav/`
`interior-cost`(top-level)의 현장·일정 데이터를 읽기 전용으로 참조합니다.

### #4. v1 자연어의 범위 — **"내 세계 안의 장소"로 한정**
- v1이 맞혀야 하는 것: **내 현장, 즐겨찾기, 최근 목적지, 내가 이미 아는 상호.**
- "성수동 조용한 카페"처럼 **세상 모든 장소를 주관적 기준으로 추천하는 것은 v2로 보류합니다.** v1에서 이걸 하려다 정확도를 잃는 것이 이 앱이 실패하는 가장 흔한 경로입니다.

## 설계 원칙

**조용히 틀린 곳으로 보내는 것이 최악이다.** AI가 목적지를 확신하지 못하면 추측해서 보내지 않는다 — 후보를 보여주고 사람이 고른다. 확신할 때만 원탭으로 끝난다. 잘못된 목적지가 차에 꽂히면 사용자는 이 앱을 두 번 다시 믿지 않는다.

## Open Questions

구현 단계에서 검증할 것들. **어느 것도 미션을 바꾸지는 않고, 방법만 바꿉니다.**

- **[리스크] 테슬라 개발자 등록 게이트** — 공식 연동에는 개발자 등록 + 내 도메인에 공개키 호스팅 + 차량에 가상키 페어링이 필요합니다. 비용은 종량제이나 **월 $10 크레딧이 있어 개인용 1대는 사실상 무료 범위**로 보입니다. `vercel.app` 서브도메인으로 도메인 검증이 통과되는지는 **구현 단계에서 확인 필요**.
  - *2021년 이전 차량이므로 서명 명령 요구는 피해 가지만, 개발자 등록 자체는 그대로 필요합니다. 구형 차량용 경로가 실제로 얼마나 단순한지는 착수 시 최신 문서로 확인해야 합니다 — 이 영역은 테슬라가 정책을 자주 바꿉니다.*
- **[리스크] 한국 주소 지오코딩** — 기존 한국 앱들이 하나같이 "주소 텍스트" 대신 좌표로 변환해 보내는 걸로 보입니다. 한글 주소를 그대로 넘기면 테슬라가 못 찾을 가능성이 높음 → 좌표 변환 경로가 필요할지 구현 단계에서 검증.
- **[연동] `interior-cost` 현장 데이터를 어떻게 읽을 것인가** — 같은 Supabase를 직접 조회할지, API를 호출할지, 주소 컬럼이 지오코딩에 쓸 만한 품질인지 착수 시 확인.
